# lp-blue — ブルー×手書き感のプレミアムLPスキル

## 起点となるリファレンス
- `reference.html` に完成実装（大人の思春期フェス／鈴木紗理奈）を保存している
- 新規LPを作る時は **必ず最初に `reference.html` を Read で全部読む** こと
- 同じデザインシステム・同じ動き・同じ構造を踏襲し、**コンテンツだけ差し替える**

## 重要: Safe Area / Layout Rules（全セクション共通）

### レイアウト原則
- **すべてのセクションは16:9標準ビューポート（1440x810以上）で、コンテンツが完全に画面内に収まること。**
- ビューポート四辺から50mm相当（約95px）の安全マージンを確保。
- セクションのpaddingを最小50px以上に設定し、各要素が画面端に接さないよう配置。
- テキスト・画像・フォーム・ボタンなど全要素が右端・下端で切れないこと。
- overflow禁止、text cutoff禁止、frame out禁止。

### テキスト制約
- 見出し・本文は1行に収まる長さに調整。複数行にわたる場合は改行位置を明示的に制御。
- 長い本文は短く簡潔に書き直す。無理に1行に詰め込まず、情報を削って収める。
- モバイル対応時も同様に、ビューポート幅に応じてテキストサイズを自動縮小するか改行を増やす。

### 生成物の品質基準
**すべての生成物は「デスクトップ・モバイル両デバイスで正常に表示でき、投影・共有に耐える品質」を基準とする。**

## デザインシステム（絶対遵守）

### カラー
```css
--blue: #1c4cc7;       /* メインブルー */
--blue-dark: #133394;
--blue-light: #7aa4ff;
--blue-pale: #e8f0ff;
--blue-faint: #f4f8ff;
--ink: #0d1a3a;        /* ダークネイビー（黒の代わり） */
--gray: #6a7592;
--bg: #fafcff;
```
- 背景は純白ではなく `#fafcff`。黒は使わず `--ink` を使う。
- アクセント色は状況に応じて拡張可（例: Schedule の時間帯カラー）

### タイポグラフィ
- 本文: `Inter` + `Noto Sans JP`（font-feature-settings: "palt"）
- 見出し（手書き風）: `Yuji Mai`（筆文字）／`Caveat`（英字手書き）
- 数字・英字強調: `Inter 800-900`

### 必須フォント読込
```html
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700;800;900&family=Noto+Sans+JP:wght@300;400;500;700;900&family=Yuji+Mai&family=Caveat:wght@500;700&display=swap" rel="stylesheet">
```

## 必須セクション構造
LPは以下の順序で構成する。各セクションは省略せず、情報がない場合は省くより**プレースホルダ文言を置く**。

1. **INTRO（全画面イントロ）** — ダークBG、タイトルが1文字ずつ落下、ブラシ下線がSVGで描画、3.4秒後にclip-pathで消える
2. **NAV** — 固定、blur、手書きロゴ + ブルー下線スワイプ
3. **HERO** — タイトル（筆文字）、ハイライト部分はSVGブラシストローク、タグ、3項目クロス（×回転アニメ）、日付・主催者
4. **CONCEPT** — セクションタイトル（手書きスクリブル）、左に4:5画像（Ken Burns）、右に本文、3本柱カード
5. **TARGET** — MAIN/SUB の2カード
6. **MARQUEE 1** — 黒地に白、キーワードが無限横スクロール
7. **SCHEDULE** — 朝→昼→日没のグラデ背景＋太陽グロー、3ブロック（朝:青／昼:橙／日没:紫）、日没に波紋アニメ
8. **CONTENTS** — 4ゾーンのカード
9. **MARQUEE 2** — 白地にブルー、逆方向
10. **OVERVIEW** — ブルー背景テーブル、数字はカウントアップ
11. **CTA** — メインメッセージ＋ボタン2つ（マグネティック）
12. **MEGA TYPE** — ダーク全画面巨大タイポ（アウトライン抜き文字を含む）
13. **FOOTER** — 3カラム、手書きロゴ、コピーライト

## 必須インタラクション（全部入れる）
- [ ] SVGブラシ下線（Hero highlight）
- [ ] セクションタイトルの手書きスクリブル（コンセプト=丸、ターゲット=二重丸、1日=波線、コンテンツ=波線、概要=丸）
- [ ] スクロールリビール（IntersectionObserver）
- [ ] スクロール進捗バー（最上部1px, 3色グラデ）
- [ ] Ken Burns（メイン画像）
- [ ] 太鼓ripple波紋（3重、1秒ずつ遅延）
- [ ] ×回転（10秒/周）
- [ ] カウントアップ（data-count属性）
- [ ] マグネティックCTA（transform translate）
- [ ] 3Dチルト（pillar & content-card、hover:noneで無効化）
- [ ] ハンバーガーメニュー（モバイル、✕変形、bodyロック）
- [ ] Heroパララックス（背景テキスト）
- [ ] BGMトグル（右下、イコライザー動的、フェードイン/アウト、セッション記憶）
- [ ] イントロローダー（文字落下＋ブラシ描画＋clip-path退場）
- [ ] Marquee無限横スクロール（2本、順/逆方向）

## モバイル対応（必須）
- ブレークポイント: `900px`（タブレット）、`640px`（スマホ）、`380px`（小型）
- タッチデバイス: `@media (hover: none)` でチルト・マグネ・ホバー演出を無効化
- reduced-motion対応
- body.intro-active で overflow:hidden
- タッチターゲットは最低 44×44
- イントロタイトルは `white-space: nowrap` で1行固定

## ワークフロー

1. **資料取得**: PDF / URL / テキストを Read
2. **情報抽出**: タイトル、タグライン、コンセプト、ターゲット、スケジュール、コンテンツ、概要項目、日付、主催者、問い合わせ先
3. **reference.html を Read（必ず全文）**
4. **新規ディレクトリ作成**: `/tmp/lp-{brand-slug}/` または `~/Desktop/{brand-slug}/`
5. **index.html 生成**: reference.html を雛形に、コンテンツだけ差し替え
6. **画像が必要なら**: ユーザーに確認 or `images/` に配置してパス参照
7. **BGMが必要なら**: ユーザーに確認して `audio/` に配置
8. **Vercelデプロイ**: 新規ディレクトリから `vercel deploy --prod --yes`（既存プロジェクトに上書きしない）
9. **URL共有**: 返却されたVercelドメインを返す

## 絶対禁止事項
- reference.htmlを読まずにスクラッチから書く（→必ずデザインが崩れる）
- カラー・フォント・セクション順序の独自変更
- ホバーエフェクト等のインタラクション削減
- 既存のVercelプロジェクトに上書きデプロイ
- 黒（#000）の使用（代わりに `--ink: #0d1a3a`）
- 純白（#fff）の背景（代わりに `--bg: #fafcff`）
- **Safe Area 無視：画面右端・下端でのテキスト切れ、画像のはみ出し、要素が画面外に出ること**
- **セクションpaddingの削減：50px未満の狭い余白で要素を配置すること**
- **overflow: hidden でテキストを隠す：見切れるくらいなら文章を短くする**

## Safe Area チェックリスト（生成・デプロイ前に必須）

LP生成後、必ず以下をチェックしてからデプロイする：

- [ ] デスクトップ表示（1440x810）：すべてのコンテンツが画面内に完全に収まっているか
- [ ] タブレット表示（900px）：レスポンシブレイアウトで、画面右端での切れがないか
- [ ] モバイル表示（640px / 380px）：文字が読める大きさに自動縮小されているか、テキストが切れていないか
- [ ] セクションタイトル（手書きスクリブル）：テキストが画面端で切れていないか
- [ ] 見出し・本文：複数行表示時の折返しが正しいか、下端でのはみ出しがないか
- [ ] ボタン（CTA）：テキストが完全に表示されているか、マグネティック効果時に画面外に出ないか
- [ ] テーブル（概要セクション）：カラムが右端で切れていないか、行高が適切か
- [ ] 画像（Ken Burns）：アスペクト比が正しく保たれているか、セクション内に収まっているか
- [ ] Marquee（無限スクロールテキスト）：テキストが左右の安全マージンを超えていないか
- [ ] モバイルメニュー：ハンバーガーメニューが右端に完全に配置されているか
- [ ] プロジェクタ投影品質：デスクトップ表示でプロジェクタに投影できる状態か
