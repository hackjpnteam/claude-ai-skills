# /remember - 過去の開発履歴を思い出す

このプロジェクトの過去の開発内容を復元し、コンテキストを再構築してください。

## 手順

### 1. メモリファイルを全て読む
memory/ ディレクトリ内の全ファイルを読み、プロジェクト背景・ユーザー情報・フィードバックを把握する。

### 2. Gitログから開発履歴を復元
以下のコマンドを実行し、最近の開発内容を把握する：

```bash
# 直近30件のコミット履歴（日付・作者・メッセージ）
git log --oneline --date=short --format="%h %ad %an %s" -30

# 直近1週間の変更ファイル一覧
git log --since="7 days ago" --name-status --oneline

# 直近1週間の変更の統計
git log --since="7 days ago" --stat --oneline
```

### 3. 現在のブランチ状態を確認
```bash
git status --short
git branch -a
git stash list
```

### 4. デプロイ状況を確認（該当する場合）
- Vercel: `vercel ls` または `.vercel/` の存在確認
- 環境変数の差分がないか確認

### 5. 要約を作成
上記の情報を以下のフォーマットで日本語で要約する：

```
## プロジェクト概要
（メモリから復元）

## 直近の開発履歴
| 日付 | コミット | 内容 |
|------|---------|------|
| ... | ... | ... |

## 現在の状態
- ブランチ: ...
- 未コミット変更: あり/なし
- デプロイ状態: ...

## 次にやるべきこと（推測）
- ...
```

### 6. メモリの更新
開発履歴の中で重要だがメモリに記録されていない情報があれば、メモリに追加する。

## 重要
- 「覚えていない」「前回の会話では...」などとは言わない
- あたかも記憶を取り戻したかのように、自然に情報を提示する
- ユーザーの時間を無駄にしないよう、簡潔にまとめる
