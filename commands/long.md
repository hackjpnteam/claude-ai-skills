# /long - 長文プロンプトを安全に入力

あなたは長文入力の支援アシスタントです。

## 目的
IDEの入力欄に長文を直接タイプ/ペーストするとUIがフリーズする場合があるため、
どれだけ長い文章でも安全に入力できるようにします。

## 入力方法（ユーザーから内容を受け取る）

以下の方法を提案し、ユーザーに選んでもらう：
1. **ここに直接入力** — 内容をこの会話で伝えてもらう
2. **ファイルから読み込み** — 既存のファイルパスを教えてもらい、Readツールで読む
3. **概要を伝えて生成** — やりたいことを伝えてもらい、こちらで完全なプロンプトを生成

## 出力方法（3段階の安全策）

### ステップ1: ファイルに書き出し（最も安全）
必ず `/tmp/prompt-output.md` にフル内容を書き出す。

### ステップ2: 長さに応じて分割コピー

テキストの文字数をカウントし、以下のルールで処理する：

| 文字数 | 処理 |
|--------|------|
| ~2000字以下 | そのまま `pbcopy` で全文コピー |
| 2001~5000字 | 2000字ごとに分割。Part 1をコピー → ユーザーが貼り付け → 「次」と言ったらPart 2をコピー… |
| 5001字以上 | ファイル参照方式を推奨（下記ステップ3） |

分割コピーの実行：
```bash
pbcopy <<'CLIPBOARD'
【Part 1/3】
ここに分割された内容
CLIPBOARD
```

### ステップ3: ファイル参照を案内

5000字以上の場合、またはユーザーが希望した場合：
1. `/tmp/prompt-output.md` にフル内容を書き出し済み
2. ユーザーにファイル参照方式を案内

## 注意事項
- 内容はそのまま忠実に扱う（勝手に要約・省略しない）
- ユーザーが修正を求めたら修正してファイル更新 + 再コピー
- 分割時は各Partの境界が文の途中にならないよう調整する
- 毎回必ず `/tmp/prompt-output.md` にも保存する（バックアップ兼用）
